# 绩优案例评测集完整溯源设计

## 1. 目标

为 `docs/新华保险AI教练问答一批绩优案例测试集.xlsx` 中的 50 组问答补充可人工审查、可机器处理的完整溯源信息。溯源目标为项目中的 `parsed/**/chunks.jsonl`，并继续展开到 chunk 内保存的原始文件、章节和定位信息。

不覆盖原始工作簿。最终产物保存为独立的 `.xlsx` 文件。

## 2. 输入与输出

### 输入

- 评测集：`docs/新华保险AI教练问答一批绩优案例测试集.xlsx`
- 溯源语料：全部 `parsed/**/chunks.jsonl`

### 输出

- 文件名：`新华保险AI教练问答一批绩优案例测试集-含完整溯源.xlsx`
- 输出位置：本次任务专属的 `outputs/019f5f79-943c-7611-b046-f9725e18b5bd/` 目录
- 原始工作簿保持不变

## 3. 工作簿结构

### 3.1 原问答工作表

保留现有工作表和前两列内容、顺序及样式，在表格右侧增加三列：

| 列名 | 含义 |
| --- | --- |
| 溯源状态 | `完整支持`、`部分支持` 或 `未定位` |
| 主 chunk_id | 与标准答案匹配度最高且能够直接提供支撑的 chunk 标识 |
| 有效溯源数 | “溯源明细”工作表中该问答对应的有效 chunk 数量 |

### 3.2 溯源明细工作表

一条问答可以对应多行溯源记录。字段如下：

| 字段 | 含义 |
| --- | --- |
| 评测行号 | 原问答工作表中的 Excel 行号 |
| 问题 | 原问题文本 |
| 匹配顺序 | 同一问答下的 chunk 支撑顺序 |
| chunk_id | `chunks.jsonl` 中的唯一 chunk 标识 |
| chunk_type | chunk 类型 |
| 案例名称 | chunk 元数据中的案例名称 |
| source_file | 原始来源文件名 |
| source_path | 原始来源相对路径 |
| section_name | 原始章节或小节名称 |
| 来源定位 | 页码、幻灯片页码、行号或标题路径的紧凑文本表达 |
| 原文摘录 | 直接支撑标准答案具体观点的最短充分原文 |
| 定位置信度 | 原 citation 的定位置信度 |
| 支撑范围及缺失说明 | 说明该 chunk 支撑答案的哪些内容，以及仍缺少哪些依据 |

同一 chunk 中存在多个有效 citation 时，每个 citation 单独形成一条明细记录；重复的 `chunk_id + source_path + locator` 记录去重。

## 4. 匹配与判定规则

### 4.1 候选召回

使用问题和标准答案共同召回候选 chunk，综合以下信号排序：

1. 标准答案中的连续原文或关键短语命中；
2. 问题与 chunk 标题、案例名称、策略名称和正文的字符 n-gram 重合；
3. 标准答案各独立观点在 chunk 正文或 citation 原文中的覆盖情况；
4. 来源文件、章节名称与问题主题的一致性。

### 4.2 有效溯源认定

只有满足以下条件之一的 chunk 才写入“溯源明细”：

- citation 原文直接包含标准答案中的关键事实或话术；
- chunk 正文对标准答案中的独立观点提供明确、可解释的支撑；
- 多个 chunk 需要共同覆盖一条长答案时，该 chunk 至少支撑其中一个可独立识别的观点。

仅主题相似、但无法支撑具体答案内容的 chunk 不纳入。

### 4.3 状态定义

- `完整支持`：标准答案的全部主要观点均有有效 chunk 和原始 citation 支撑。
- `部分支持`：至少一个主要观点有明确支撑，但答案存在缺项、扩写、绝对化表述或无法找到对应原文的部分。
- `未定位`：没有找到能够直接支撑标准答案主要观点的 chunk。

溯源状态只评价“答案是否有源文依据”，不代表答案在法律、监管或事实层面正确。

## 5. 格式设计

- 延续原工作簿的表头风格，不重排前两列。
- 新增列启用自动换行并设置合理列宽，避免单元格内容被截断。
- 原问答工作表冻结首行并保留现有筛选能力。
- “溯源明细”工作表使用一致的标题行、筛选和冻结首行。
- `溯源状态` 使用条件格式：完整支持为绿色、部分支持为黄色、未定位为红色。
- 长原文摘录限制为最短充分片段，不整段复制 chunk 正文。

## 6. 异常与边界处理

- chunk 无 citation：可以保留 chunk 级溯源，但必须标记“缺少原始 citation 定位”。
- citation 无精确 locator：使用 `section_name + source_file`，定位置信度按原数据保留。
- 同一问答命中重复或高度相似 chunk：按唯一 chunk 和唯一原始 locator 去重。
- 原始答案本身不完整或存在风险表述：不修改标准答案，仅在“支撑范围及缺失说明”中如实记录。
- 不在工作簿中写入客户姓名、电话、证件号、保单号等新的个人敏感信息。

## 7. 验证标准

1. 原问答工作表仍为 50 组问答，前两列内容与原文件逐单元格一致。
2. 每条问答都有溯源状态；`有效溯源数` 与明细表记录一致。
3. 每个主 `chunk_id` 均能在某个 `parsed/**/chunks.jsonl` 中找到。
4. 对 `完整支持`、`部分支持`、`未定位` 分别抽查代表性记录，核对原文与定位。
5. 工作簿无公式错误，表头、长文本、筛选和冻结窗格正常。
6. 对两个工作表分别进行完整视觉渲染检查，确认没有严重截断、重叠或不可读区域。
