# 会话侧栏与本地持久化设计

## 背景

当前 Web 前端是单页 React 应用，问答轮次存放在 `App` 组件的 `turns` 状态中。刷新页面会丢失历史；后端没有会话相关 API。用户希望在左侧增加会话列表，支持选择会话和创建新会话，并要求刷新后会话继续保留，除非手动删除。

## 目标

- 在左侧新增会话列表。
- 支持创建新会话，并立即切换到新会话。
- 支持选择已有会话，问答区和右侧溯源详情随当前会话切换。
- 支持删除会话；删除是移除持久化数据的唯一入口。
- 刷新页面后恢复会话列表、当前会话和每个会话的问答历史。
- 保持现有问答、流式回答、引用选择、bad case 反馈和参数校验行为。

## 非目标

- 不新增后端会话 API。
- 不做跨浏览器或跨设备同步。
- 不引入 IndexedDB、数据库或认证体系。
- 不改变问答接口请求结构。

## 推荐方案

使用前端 `localStorage` 持久化会话数据。当前问答状态完全由前端持有，`localStorage` 可以在不扩大后端范围的情况下满足刷新后保留的需求。

## 数据模型

新增前端类型 `ChatSession`：

```ts
type ChatSession = {
  id: string;
  title: string;
  created_at: string;
  updated_at: string;
  turns: ChatTurn[];
};
```

新增本地存储形状：

```ts
type StoredChatSessions = {
  version: 1;
  active_session_id: string;
  sessions: ChatSession[];
};
```

存储 key 使用固定命名，例如 `xhbx-rag.chat-sessions.v1`。如果没有可用存储、数据损坏或会话列表为空，前端创建一个空白会话作为恢复兜底。

## 交互设计

左侧侧栏：

- 顶部显示“会话”和“新会话”按钮。
- 列表项显示会话标题、更新时间和问答轮次数。
- 当前会话列表项有选中状态。
- 每个列表项提供删除按钮。删除当前会话后切到剩余会话中最近更新的一条；删除最后一个会话时自动创建空白会话。

中间问答区：

- 显示当前会话的问答轮次。
- “清空”只清空当前会话的 `turns`，不删除会话记录。
- 提交第一个问题后，如果当前会话仍是默认标题，用问题前若干字符生成标题。
- 流式回答过程中的步骤、增量答案、最终答案和错误都写回当前会话。

右侧溯源区：

- `latestResponse` 和检索证据从当前会话 `turns` 计算。
- 切换会话时，如果新会话有最新回答，默认选中该回答的第一个引用；如果没有引用，清空引用详情。
- 删除或清空会话时清空引用详情和 Finder 提示。

## 错误处理

- `localStorage` 读取失败、JSON 解析失败或 schema 不完整时，回退到单个空会话。
- `localStorage` 写入失败时不阻断问答流程；前端状态仍继续工作。
- 删除会话需要阻止事件冒泡，避免误触发选择会话。

## 测试计划

前端测试采用现有 Vitest 和 Testing Library：

- 刷新前已存在的会话会从 `localStorage` 恢复并显示历史问答。
- 点击“新会话”会创建并切换到空会话，原会话历史保留。
- 选择不同会话会切换问答历史和右侧引用上下文。
- 删除会话会从列表和 `localStorage` 移除；删除最后一个会话会保留一个空白会话。
- 提交第一个问题后，会话标题会从问题生成并持久化。
- 现有提交问答、Enter 发送、Shift+Enter 换行、参数校验和清空按钮行为保持通过。

## 实现范围

预计修改：

- `web/src/types.ts`：增加会话类型。
- `web/src/App.tsx`：引入会话状态、持久化读写、左侧会话栏和切换逻辑。
- `web/src/styles.css`：新增三栏布局和会话列表样式。
- `web/src/App.test.tsx`：增加会话行为测试并调整清空按钮断言。

后端 Python 文件和 API 客户端无需修改。
