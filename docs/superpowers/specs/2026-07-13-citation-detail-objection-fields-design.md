# 引用明细异议字段精简设计

## 目标

精简 Web 问答界面右侧“引用明细”，让用户优先看到异议处理结论，减少标签与其他结构化字段造成的视觉干扰。

## 展示规则

- 完全移除标签展示区域，不显示合规标签、普通命中标签或标签提权。
- 正文框只显示“客户异议”“异议诊断”“推荐回应”，并按该顺序排列。
- 不在正文框显示其他结构化字段、纯文本或案例原文字段。
- 三个目标字段均不存在时，显示“暂无异议处理内容”。
- 来源引用、文件信息、位置、定位、原文摘录、Finder 按钮和“应该用/不该用”反馈保持现状。

## 实现边界

仅修改前端引用明细的渲染逻辑，不修改后端接口、检索结果结构或入库数据。字段筛选在展示层完成，避免影响其他数据消费者。

## 测试

- 验证所有标签信息均不渲染。
- 验证正文只渲染三个目标字段并保持固定顺序。
- 验证没有目标字段时显示空状态，不回退展示原始全文。
- 保留既有来源引用和反馈交互测试，确保相关功能不受影响。
