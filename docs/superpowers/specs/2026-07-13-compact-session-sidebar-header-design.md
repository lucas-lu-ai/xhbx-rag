# 会话侧栏头部紧凑布局设计

## 目标

在隐藏文档入库入口后，重新整理知识问答侧栏头部：将知识问答入口居中，删除“会话 / 问答会话”标题，并让“新会话”和“批量执行”同排等宽显示。

## 设计

- `WorkspaceNav` 改为单项居中布局；知识问答按钮保持约原单列宽度，不拉伸占满整行。
- `SessionSidebar` 删除 `session-header` 内的标题文案节点，仅保留操作按钮区域。
- 两个操作按钮在同一行等分可用宽度，间距使用现有 8px 设计节奏。
- 操作按钮继续保持至少 44px 高度、Lucide 图标、文字标签、悬停态和键盘焦点态。
- 在 `max-width: 920px` 响应式规则下仍保持同排，通过弹性宽度避免横向溢出，不再把两个按钮强制改为纵向堆叠。
- 不修改历史会话列表、按钮行为、文档入库页面功能或知识问答主内容。

## 测试

- 组件测试断言“会话”和“问答会话”文案不再出现。
- 样式测试断言工作台导航使用居中布局，操作区不换行且两个按钮弹性等宽、最小高度保持 44px。
