# 引用详情横向溢出调整设计

## 目标

右侧“引用明细”面板只允许纵向滚动，不因来源名称、文件路径或摘录内容过长而产生横向滚动。

## 界面调整

1. 来源引用胶囊保持单行显示，宽度不超过详情面板；超出部分以省略号截断。胶囊的 `title` 继续保留完整来源信息，鼠标悬停时可查看全文。
2. 原有“位置”和“定位”两个信息框合并为一个“位置与定位”信息框，按“位置 · 定位状态”的顺序展示；缺失的部分不输出多余分隔符，两者都缺失时显示“未提供”。
3. 删除“在 Finder 中显示文件”按钮，以及只为该操作服务的成功或失败提示和前端处理逻辑。
4. 详情面板及其内容容器限制横向溢出；文件名、位置与定位、原文摘录等正文内容允许在任意长字符串处换行。

## 保持不变

- 点击来源胶囊切换当前引用的交互不变。
- 来源胶囊的选中态、展开/收起多条来源功能不变。
- 客户异议、异议诊断、推荐回应及两维度反馈流程不变。
- 右侧面板仍可纵向滚动。

## 验收与测试

- 长来源名称只占一行，呈现省略号，不撑宽容器。
- 页面不再显示 Finder 按钮及对应状态提示。
- “位置与定位”正确组合完整、部分缺失及全部缺失的数据。
- 现有引用切换和反馈测试继续通过，前端测试与生产构建通过。
