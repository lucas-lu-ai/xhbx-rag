# Web 问答检索数量配置化设计

## 目标

隐藏知识问答页面的“召回 / 引用”输入控件，改由项目根目录 `.env` 统一配置单次问答与批量执行的检索数量。

## 配置

新增两个环境变量：

```env
WEB_RETRIEVAL_TOP_N=20
WEB_RETRIEVAL_TOP_K=5
```

- 未配置时默认值分别为 `20` 和 `5`，保持当前行为。
- `WEB_RETRIEVAL_TOP_N` 必须为 `1–100` 的整数。
- `WEB_RETRIEVAL_TOP_K` 必须为 `1–20` 的整数。
- `WEB_RETRIEVAL_TOP_K` 不能大于 `WEB_RETRIEVAL_TOP_N`。
- 非法配置使 Web 状态返回明确的安全错误，不静默回退到其他值。

## 数据流

1. 后端配置模块读取并校验 Web 检索数量。
2. `/api/status` 返回安全的 `web_retrieval_top_n` 与 `web_retrieval_top_k` 数值。
3. `App` 从状态响应取得数值并传给单次问答与批量创建组件。
4. `ChatView` 不再维护可编辑的数量状态，也不渲染两个数字输入框；创建 turn 和发送请求时直接使用传入值。
5. `BatchCreateView` 解析粘贴文本、CSV 或 XLSX 时使用相同数值写入每一行任务。

## 兼容范围

- 保留 Web API 请求中的 `top_n` 与 `top_k` 字段及既有校验，避免破坏历史记录、bad case、批量存储和外部调用。
- MCP、CLI 和 A2A 的显式参数行为不受 Web 配置影响。
- 配置加载失败时状态接口仍提供现有安全错误结构，并使用默认数值填充响应字段，确保前端能够稳定渲染错误提示。
