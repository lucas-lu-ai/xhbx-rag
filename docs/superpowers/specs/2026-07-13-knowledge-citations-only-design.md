# 知识引用仅展示模型实际引用设计

## 目标

将问答结果中的“检索证据”改为“知识引用”，只展示回答模型实际选择的引用证据；同步优化右侧详情，使用户看到引用序号和知识名称，而不是内部知识类型。

单次问答和批量详情共用同一套展示与交互规则。

## 引用判定

- 继续使用答案级 `citations` 中的 `selected === true` 与 `evidence_index` 判断模型实际引用。
- `evidence_index` 是 `retrieval_evidences` 的 1-based 原始位置。
- `selected === false` 的兜底引用，以及没有 `selected/evidence_index` 标记的旧数据，不作为知识引用展示。
- 没有实际引用时，不渲染知识引用折叠区，也不自动选中任何召回证据。

## 列表展示

- 折叠区可见标题与无障碍名称统一由“检索证据”改为“知识引用”。
- 标题计数只显示实际引用数量，例如“知识引用 · 2 条”，不再同时显示召回总数和“答案引用”统计。
- 列表只渲染实际引用的证据，并按它们在 `retrieval_evidences` 中的原始顺序排列。
- 可见序号重新连续编号为 1、2、3；内部选择 key、右侧明细和反馈操作仍使用原始证据索引。
- 保留列表中的知识类型小标签和重排分；移除每行重复的“答案引用”徽标。

## 右侧引用明细

- 右侧区域标题由“证据明细”改为“引用明细”。
- 点击知识引用后，详情顶部显示 `引用N：知识名称`，例如 `引用1：【张丽】专业第一`。
- 知识名称沿用列表当前的 metadata 格式化结果；缺少 metadata 时回退为“未命名知识”。
- 详情顶部不再显示 `证据 N · 异议处理`，也不显示重复的“答案引用”徽标。
- metadata 已进入详情主标题后，不再在下一行重复展示。
- 重排分、正文、标签、来源文件、原文摘录与“应该用 / 不该用”反馈能力保持不变。
- 右侧未选择提示改为“点击一条知识引用查看明细。”。

## 索引与数据完整性

前端构造引用视图条目，每项同时保存：

- `evidence`：原始证据对象；
- `evidenceIndex`：在完整召回数组中的 0-based 原始索引；
- `displayIndex`：过滤后的 0-based 可见序号。

列表显示使用 `displayIndex`，选择 key、证据明细、反馈回调使用 `evidenceIndex`。这样即使实际引用的是原始第 2、5 条，界面仍显示“引用1、引用2”，点击和反馈也会命中原始第 2、5 条。

展示过滤不修改 API 响应，也不裁剪 bad case 请求中的 `retrieval_evidences`。完整召回数据继续用于诊断、相关话术查找和反馈记录。

## 测试

- `EvidenceList` 测试覆盖标题改名、计数简化、仅显示实际引用、连续可见编号、原始 key 映射和徽标移除。
- `EvidenceDetail` 测试覆盖“引用明细”语义、`引用N：知识名称` 标题、metadata 回退和重复信息移除。
- `firstEvidenceKey` 测试覆盖优先选择第一个实际引用，以及无实际引用时返回空。
- 单次问答和批量详情测试覆盖共享组件中的新标题、过滤结果、点击明细和反馈仍指向正确原始证据。
- 运行完整前端测试与生产构建。
