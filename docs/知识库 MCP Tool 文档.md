# 知识库 MCP Tool 文档

# 概述

本文档面向下游 Agent 开发者，说明知识库 MCP Server 暴露的 Tool 接口、参数及返回结构。

所有 Tool 返回统一包装为 `McpResponse<T>`：

- 成功：`{"success":true,"data":...}`

- 失败：`{"success":false,"errorCode":"...","errorMessage":"..."}`

# Tool 列表

## kb\_list\_knowledge\_bases

**用途：**列出当前用户有权限查阅的知识库。

**调用时机：**当用户询问有哪些知识库可用，或需要先确认可检索范围时调用。

### 参数

无参数。

### 返回 data

```json
[{"kbId": 1,"name": "产品知识库","description": "..."}]
```

## kb\_search\_knowledge

**用途：**在指定知识库中统一检索 QA、文档切片、知识点。

**调用时机：**当用户提出具体问题、需要召回知识内容片段时调用。调用前应先使用 `kb_list_knowledge_bases` 获取可见知识库 ID。

### 参数

|字段|类型|必填|说明|
|---|---|---|---|
|query|string|是|检索 query，支持自然语言问题或关键词|
|kbId|long|是|目标知识库 ID，必须来自 kb\_list\_knowledge\_bases 返回的可见知识库|
|knowledgeTypes|List\<string\>|否|知识形态：QA / SLICE / KNOWLEDGE\_POINT，默认 \[QA, SLICE, KNOWLEDGE\_POINT\]|
|retrievalMode|string|否|检索方式：SPARSE / VECTOR / HYBRID，默认 HYBRID|
|hybridWeights|object<br>|否|混合检索权重，仅 HYBRID 生效。sparseWeight 为全文权重，vectorWeight 为语义权重，默认均为 0\.5。比如<br>"hybridWeights": \{<br>"vectorWeight": 0\.7,<br>"sparseWeight": 0\.3<br>\},|
|topK|int|否|返回数量，默认 10，最大 50|
|includeDetails|boolean|否|是否返回完整切片结构，默认 false。省略或设为 false 时返回精简结构；设为 true 时返回原完整结构|

### 返回 data

`includeDetails` 省略或为 `false` 时，每条结果只返回完整正文 `content`，以及第一条引用的 `source_path` 和 `filename`。正文或引用字段不存在时，对应字段为空字符串：

```json
{
  "success": true,
  "data": [
    {
      "content": "先共情客户对预算的顾虑，再澄清其保障需求。",
      "source_path": "案例A/预算异议.txt",
      "filename": "预算异议.txt"
    }
  ],
  "errorCode": null,
  "errorMessage": null
}
```

`includeDetails` 为 `true` 时返回原完整切片结构。每条结果包含公共字段 `id`、`knowledgeType`、`score`、`tags`，以及按形态区分的详情字段：
